PROJECT NAME
MyTube (simple YouTube-style Flutter learning app)

GOAL OF THIS PROJECT
This is a beginner-friendly mobile app for practice in labs.
It demonstrates:
1) Simple page navigation
2) Basic sign up and login flow
3) Passing data to widgets
4) Displaying a list of video cards
5) Showing a video detail screen

IMPORTANT DESIGN IDEA
The app uses ONLY beginner-friendly Flutter widgets so any file
can be rewritten from memory. No plugins, no complex patterns.


==================================================
1) FULL APP FLOW (VERY SIMPLE MAP)
==================================================

App starts -> Welcome screen
Welcome screen has 2 buttons:
  1) Sign Up  -> save email/password in memory -> go to Home
  2) Log In   -> check entered email/password against saved values -> if correct go to Home

Home screen:
  1) Shows a list of sample videos
  2) Each item is a Card with an Image thumbnail and a ListTile
     (title, channel, views, play icon)
  3) Tap a card -> open Video Player screen with that video's title
  4) Logout icon -> clears navigation stack and goes back to Welcome

Video Player screen:
  1) Shows a large play icon
  2) Shows the selected video title
  3) Has a "Go Back" button that returns to the Home screen


==================================================
2) FILE-BY-FILE EXPLANATION
==================================================

----------------------------------
FILE: lib/main.dart
----------------------------------
Purpose:
Main entry point of the app.

Code meaning:
1) import 'package:flutter/material.dart';
   - Imports Flutter Material widgets (Scaffold, AppBar, Text, etc).
2) import 'welcome_page.dart';
   - Imports first screen widget.
3) void main() { runApp(MyApp()); }
   - App starts here.
   - runApp mounts root widget into screen.
4) class MyApp extends StatelessWidget
   - Root widget does not hold mutable state.
5) MaterialApp(...)
   - Sets app title, hides debug banner, and sets home page.
6) home: WelcomePage()
   - First visible screen is Welcome page.


----------------------------------
FILE: lib/auth.dart
----------------------------------
Purpose:
Tiny in-memory credential storage.

Code meaning:
1) String savedEmail = '';
2) String savedPassword = '';

These two global variables hold sign up values while app is running.
No database, no encryption, no persistence after app closes.


----------------------------------
FILE: lib/welcome_page.dart
----------------------------------
Purpose:
Landing screen with app logo/name and navigation to auth pages.

Widgets used:
  Scaffold, Center, Column, Icon, Text, SizedBox,
  ElevatedButton, TextButton, Navigator

Code meaning:
1) Imports signup and login pages.
2) class WelcomePage extends StatelessWidget
   - UI only, no mutable state.
3) Scaffold
   - Basic page structure.
4) Column centered on screen with:
   - Red play icon (Icons.play_circle_fill)
   - App name text "MyTube"
   - Subtitle text
   - Sign Up ElevatedButton (red)
   - Log In TextButton (red text)
5) Sign Up button onPressed:
   Navigator.push(...SignUpPage())
   - Opens sign-up page on top of stack.
6) Log In button onPressed:
   Navigator.push(...LoginPage())
   - Opens login page on top of stack.


----------------------------------
FILE: lib/signup_page.dart
----------------------------------
Purpose:
Collect email and password and store them in memory.

Widgets used:
  Scaffold, AppBar, Padding, Column, TextField, SizedBox,
  Text (error), ElevatedButton, Navigator

Code meaning:
1) class SignUpPage extends StatefulWidget
   - Uses state because text fields/controllers are dynamic.
2) TextEditingController emailController/passwordController
   - Read typed input from text fields.
3) String errorMessage = ''
   - Holds the inline error text (shown below fields when invalid).
4) signUp() method:
   - Reads input values.
   - Checks if both fields are non-empty.
   - If valid: assigns values to savedEmail/savedPassword from auth.dart.
     Then Navigator.pushReplacement(...HomePage())
     (replaces page so user cannot go back to sign-up with back press).
   - If invalid: setState sets errorMessage to show a red Text below fields.
5) build() returns Scaffold with:
   - AppBar title "Create Account"
   - Email TextField
   - Password TextField (obscureText: true)
   - Red Text showing errorMessage (empty when no error)
   - Create Account ElevatedButton that calls signUp


----------------------------------
FILE: lib/login_page.dart
----------------------------------
Purpose:
Validate typed credentials against saved credentials.

Widgets used:
  Scaffold, AppBar, Padding, Column, TextField, SizedBox,
  Text (error), ElevatedButton, Navigator

Code meaning:
1) StatefulWidget with two TextEditingControllers.
2) String errorMessage = ''
   - Holds the inline error text.
3) login() method:
   - Reads input values.
   - Compares entered email/password with savedEmail/savedPassword.
   - If both match: Navigator.pushReplacement(...HomePage())
   - Else: setState sets errorMessage to "Wrong email or password".
4) build() is similar to SignUp page:
   - Email and password TextFields
   - Red Text showing errorMessage
   - Log In button triggers login().


----------------------------------
FILE: lib/home_page.dart
----------------------------------
Purpose:
Show feed of videos and open player page.

Widgets used:
  Scaffold, AppBar, Icon, Text, SizedBox, IconButton,
  ListView.builder, Card, Column, Image.network, ListTile, Navigator

Code meaning:
1) Imports:
   - welcome_page.dart (for logout destination)
   - video_player_page.dart (for detail screen)
2) class HomePage extends StatelessWidget
3) videos list:
   - A static List<Map<String, String>>
   - Each item has: title, channel, views, thumbnail (URL)
4) AppBar:
   - Left: Row with play icon + "MyTube" text
   - Right action: logout IconButton
5) Logout action uses Navigator.pushAndRemoveUntil(...WelcomePage(), false)
   - Sends user to welcome page and clears navigation history.
6) body uses ListView.builder
   - Builds a Card for each video item.
7) Each Card contains:
   - Image.network (thumbnail, 180px tall) with error fallback
   - ListTile with:
       leading: play Icon
       title: video title Text
       subtitle: channel + views Text
       onTap: opens VideoPlayerPage(title: ...)


----------------------------------
FILE: lib/video_player_page.dart
----------------------------------
Purpose:
Show the selected video's title and a back button.
(No actual video playback — keeps things beginner-friendly.)

Widgets used:
  Scaffold, AppBar, Center, Column, Icon, Text, SizedBox,
  ElevatedButton, Navigator

Code meaning:
1) StatelessWidget - no state needed.
2) Input field:
   final String title;
   - Receives the selected video title from Home page.
3) build() returns Scaffold with:
   - AppBar title "Now Playing" (red background)
   - Body: centered Column with:
       large play Icon (red, size 100)
       Text showing the video title
       Text "Video is playing..."
       ElevatedButton "Go Back" that calls Navigator.pop(context)

==================================================
4) HOW TO REWRITE THIS PROJECT QUICKLY IN LAB
==================================================

If teacher deletes a file, rebuild in this exact order:

1) Create main.dart
   - MaterialApp -> home WelcomePage.

2) Create auth.dart
   - savedEmail and savedPassword globals.

3) Create welcome_page.dart
   - App icon/name + ElevatedButton (Sign Up) + TextButton (Log In).

4) Create signup_page.dart
   - Two TextFields, String errorMessage, ElevatedButton.
   - Save values to auth.dart, navigate to Home.

5) Create login_page.dart
   - Compare values with auth.dart, navigate to Home if valid.
   - Show errorMessage Text if invalid.

6) Create home_page.dart
   - Hardcoded videos list.
   - ListView.builder with Card + Image.network + ListTile.
   - Logout IconButton in AppBar.

7) Create video_player_page.dart
   - StatelessWidget that receives a title String.
   - Icon + Text + ElevatedButton (Go Back -> Navigator.pop).

8) In pubspec.yaml: no extra dependencies to add.

9) Run:
   flutter pub get
   flutter run -d chrome   (or your device)


==================================================
5) KNOWN LIMITATIONS (INTENTIONAL FOR SIMPLICITY)
==================================================

1) Auth is not real security (plain in-memory strings).
2) Credentials reset when app closes.
3) No backend/database.
4) No search, no comments, no subscriptions.
5) No real video playback - replaced with a static detail screen.
6) Video titles and thumbnails are hardcoded sample data.


==================================================
6) ONE-SCREEN MEMORY CHEAT SHEET
==================================================

Flow:
  main -> welcome -> signup/login -> home(list) -> video detail

Coding pattern:
1) Inputs with TextEditingController
2) Validate in button function, show error with setState + Text widget
3) Navigate with Navigator.push / pushReplacement / pop
4) Use Card + ListTile for list items
5) StatefulWidget only when controller or error state is needed

This is enough to recreate the full app quickly.
